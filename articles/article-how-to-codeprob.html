Getting Started with CodeProb

Youâ€™ve seen what CodeProb is all about, but now the question arises: How do you actually start CodeProbing?

CodeProb is a place where programmers like you share insights on the latest trends, break down complex concepts, and challenge others with programming problems. Your contributions help those just starting out or looking to develop new skills.

Because CodeProb is built entirely on GitHub Pages (making it fast, free, and static), the process might seem a little different than usual web forms. But don't worry it's simpler than it looks. Let's walk through it together!

1. Creating Your Profile

To do anything on CodeProb like creating an Article, Concept, Problem, or Profile we use the Writer Interface.

Step 1: Generate your Profile HTML

  • Open the Writer and select the Profile tab.
  • Fill in your details (Username, Display Name, Bio, etc.).

    • > Note: Choose your username carefully (e.g., axshul) as it will be part of your URL.

  • Once satisfied with the preview, click Export HTML.
  • An index.html file will automatically download to your system.

    • Step 2: Fork and Upload

    Now we need to get that file onto the internet.

  • Fork the main CodeProb Repository.
  • Inside your forked repo, open the profiles folder.
  • Create a new folder that matches the username you entered in the Writer (e.g., profiles/axshul/).
  • Upload the index.html file you downloaded into this new folder.

    • Step 3: Registering Your Profile

    There is one last crucial step to make your profile visible to the world: indexing it in the profiles.json file.

  • In the root of your repository, open the file named profiles.json.
  • Locate the "profiles": [...] list.
  • You simply need to append your details to this list.

    • For example, if my username is axshul, I would add a new block like this:

    {
        "profiles": [
            {
                "username": "codeprob team",
                "displayName": "CodeProb Team",
                "dateAdded": "2026 01 01"
            },
            {
                "username": "axshul",         // <    Your new entry
                "displayName": "Anshul Namdev",
                "dateAdded": "2026 01 01"
            }
        ]
    }
    What to do: Pay attention to the syntax! Add a comma , after the previous item, then create a new object { } containing your username, displayName, and dateAdded.

    Step 4: Go Live!

    Once you've added your folder and updated the JSON:

  • Commit your changes.
  • Create a Pull Request to the main CodeProb Repository.

    • We will validate your request and merge it. Shortly after (usually 2 5 minutes), your profile will be live at:

    https://codeprob.github.io/CodeProb/profiles/<your username>

    Voila! You are live!

    The main advantage of creating a profile is attribution. When you write an Article, Concept, or Problem, simply use your Display Name as the author. The system will automatically link your name to your profile page, allowing readers to discover you and see all your contributions in one place!

    2. Writing Your First Article

    Got something to say? Whether it's a tutorial, an opinion piece, or a deep dive into a new tech stack, Articles are the way to go.

    Step 1: Write & Export

  • Head back to the Writer and click the Article tab.
  • Unleash your creativity! The editor supports full Markdown, so you can add code blocks, bold text, images, and links.
  • Click Export HTML to get your file (e.g., article my first post.html).

    • Step 2: Upload

  • In your forked repo, navigate to the articles folder.
  • Upload your exported HTML file here.

    • Step 3: Indexing

    Just like the profile, we need to tell CodeProb about your new article. Open contributor-config.json in the root directory and find the "articles" array.

    Add your entry:

    

    // Inside "description": { "content": { "articles": [ ...
    {
        "filename": "article my first post.html",
        "title": "My Awesome First Article",
        "tags": ["Tutorial", "Beginner", "JavaScript"],
        "author": "Anshul Namdev",
        "dateAdded": "2026 01 02"
    }

    3. Adding a Concept

    CodeProb Concepts are wiki style explanations of programming terms (e.g., "Recursion", "Big O Notation", "Closures"). They should be educational and objective.

    Step 1: Define & Export

  • Select the Concept tab in the Writer.
  • Fill in the Category (e.g., Algorithms) and Complexity (e.g., Beginner).
  • Provide a clear Overview and Detailed Explanation with code examples.
  • Click Export HTML to download your concept file.

    • Step 2: Upload & Index

  • Upload the file to the concepts folder in your repo.
  • Open contributor-config.json and add to the "concepts" array:

    • {
        "filename": "recursion explained.html",
        "title": "Understanding Recursion",
        "category": "algorithms",
        "complexity": "intermediate",
        "author": "Anshul Namdev",
        "dateAdded": "2026 01 02"
    }

    4. Creating a Problem

    Challenge the community! Problems are the heart of CodeProb.

    Step 1: Create & Export

  • Select the Problem tab.
  • Define the Difficulty, Topics, and provide a clear Description.
  • Crucial: Provide clear Input/Output examples so users know exactly what to build.
  • Click Export HTML.

    • Step 2: Upload & Index

  • Upload the file to the problems folder.
  • Open contributor-config.json and add to the "problems" array:

{
    "filename": "array sum problem.html",
    "title": "Sum of Array Elements",
    "difficulty": "easy",
    "topics": ["arrays", "math"],
    "author": "Anshul Namdev",
    "dateAdded": "2026 01 02"
}

Final Step: Submit!

Once you've uploaded your content files and updated the contributor config.json, simply Commit and Push your changes, then open a Pull Request.

Reviewers will check your submission, and once merged, your contribution will be part of the CodeProb learning ecosystem forever. Happy CodeProbing!
